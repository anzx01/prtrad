# Polymarket Tail Risk Web 应用 Backlog（M1-M2）
## 可直接分发给 Codex / Claude Code

---

# 1. 使用说明

本 Backlog 对应里程碑：

- `M1`：数据基础与质量控制可用
- `M2`：标签分类与人工审核闭环可用

每个任务都包含：

1. 任务编号
2. 负责人泳道
3. 优先级
4. 依赖
5. 任务描述
6. 交付物
7. 验收标准（DoD）

优先级定义：

- `P0`：阻塞主链路，必须优先完成
- `P1`：关键功能，紧随 P0
- `P2`：增强项，不阻塞 M1-M2 验收

泳道定义：

- `DATA`：数据与任务线
- `RISK`：业务与风控线
- `WEB`：前端线
- `PLAT`：平台与运维线

# 2. M1 Backlog（数据基础与质量控制）

- 里程碑：`M1`
- 泳道：`PLAT`

## M2-002 自动分类引擎 v1

- 优先级：`P0`

1. 每个可处理市场都有分类结果或明确失败原因
2. 结果包含类别、因子、置信度、规则版本

- 验收标准：
1. 每个分类结果具备两类评分
2. 低于阈值的市场被标记为 `ReviewRequired` 或拒绝
3. 评分依据可追溯

# 6. 可直接下发给编码代理的任务指令模板

给 Codex/Claude Code 下发单个任务时，建议使用如下模板：

1. 任务编号：例如 `M2-004`
2. 任务目标：复制本 Backlog 对应描述
3. 写入范围：明确允许修改的目录
4. 输入契约：引用本 Backlog 的输入依赖
5. 输出契约：引用本 Backlog 的交付物
6. 验收标准：逐条对照 DoD
7. 禁止事项：不修改无关模块，不跳过审计与版本字段
