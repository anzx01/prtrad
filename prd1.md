**Polymarket Tail Risk NO 策略 V3 PRD**
**（可生存、可验证、可执行版本：强化概率校准 + 严格风控 + 数据驱动黑天鹅防护）**

---

### 1. 核心升级目标（从第一性原理出发）

预测市场的本质是 **信息聚合与定价** ，但人类行为、流动性摩擦、结算机制会引入系统性偏差。V3 的唯一目标仍是：

> **在充满不确定性与黑天鹅的世界中，系统长期存活并实现可持续正期望收益。**

相比 V2，V3 重点解决以下根本问题：

* 数据稀疏与校准不稳定（历史样本不足、动态偏差）
* 分类边界模糊与执行难度
* 流动性与执行摩擦（薄市场、滑点、部分成交）
* 结算模糊与 UMA 争议风险（即使白名单类别也可能受影响）
* 尾部相关性与连环风险（正常相关性 vs 危机相关性）
* 机会稀缺与资金利用率低
* 回测可信度（避免未来函数、过拟合）

 **第一性原理** ：我们不预测具体事件，只利用可验证的 **历史统计边缘** （市场定价偏差）、**明确可执行规则**和**多层约束**来限制下行，同时让系统在大多数时间“什么都不做”。

---

### 2. 策略本质（重新定义）

V2：只在“被错误高估的小概率事件”中卖 NO，并动态控制暴露。

 **V3** ：

> **利用白名单类别中可验证的历史定价偏差（实际发生率 vs 市场隐含概率），通过严格筛选、实时校准、流动性过滤和极端风控，仅在正边缘 + 可控风险时小仓位参与。**

核心是 **生存优先、边缘次之** ：宁可长期空仓，也不冒不可量化风险。

---

### 3. 三层策略结构（强化版）

#### Layer 1：候选筛选（更严格过滤）

基础条件（必须全部满足）：

* NO 价格 ≥ 0.97（默认阈值，后续可通过回测优化）
* 剩余时间 ≤ 5 天（比 V2 的 7 天更保守，减少暴露时间）
* 市场有**明确、可客观验证**的结算规则（包含“according to official source / oracle / API / data feed”等关键词）
* 最低流动性阈值：当前深度（bid/ask）支持至少 X USDC 成交（初始建议单仓规模的 5-10 倍，实时检查）
* 排除明显争议历史类似市场

#### Layer 2：事件分类（自动化 + 人工审核结合）

 **白名单类别** （仅允许）：

* **Numeric** ：有可靠价格 feed（如 BTC/ETH 等主流资产官方价格）
* **Time** ：有明确时间戳或截止日期，可通过公开记录验证
* **Statistical** ：基于官方统计数据或可重复计算的结果

 **灰名单** （允许极小测试仓位，单独跟踪）：

* 边界清晰的混合类别，需人工审核 + 额外校准

 **严格禁止** ：

* Macro / 政治 / 人物行为（Person）
* War / Disaster / 地缘事件
* Uncertain / 规则模糊 / 依赖主观解读
* 历史曾出现 UMA 重大争议的类似结构

 **分类实现** ：规则-based + 简单 NLP（关键词匹配 + 正则），初期人工复核所有候选，成熟后全自动。增加“明确性评分”（0-100 分），低于 85 分禁止。

#### Layer 3：概率校准与决策（动态 + 置信区间）

 **核心思想** ：市场价格 ≠ 真实概率。利用历史分桶统计偏差，但必须处理样本不足问题。

 **实现方式** ：

* 使用 Polymarket Gamma API + CLOB prices-history + Data API 构建可回放数据集（market_id、question、category、open/close_time、price_series、resolution）。
* 分桶（按 NO 价格）：
  * 0.97–0.98
  * 0.98–0.99
  * 0.99+
* 按类别分别统计：样本数、实际 YES 发生率、置信区间（使用 Wilson score 或 bootstrap）。
* **决策规则** （保守版）：
* 仅当 **实际发生率 < 市场隐含概率（1-NO）** 且  **样本数足够** （每桶每类别至少 N=30，视回测调整）且 **置信区间支持正边缘** 时，才允许交易。
* 样本不足或置信区间过宽 → 禁止该桶/类别。
* 采用滚动窗口校准（过去 12-24 个月数据），每周或每月更新一次，避免静态偏差。
* 额外：监控全平台或同类别近期偏差趋势，若整体偏差反转，临时提高阈值。

---

### 4. 相关性与仓位风险控制（最重要层）

 **问题** ：尾部事件往往同时发生（correlation in tails）。

 **解决方案** ：

1. **分类限仓** ：

* 每个白名单子类最大 25%（比 V2 更严）
* 单一类别绝不超过 35%

1. **主题/资产限仓** ：

* 同一资产（如 BTC 相关）、同一时间窗口、同一宏观主题 → 每个主题最大 8-10%

1. **相关性矩阵（升级版）** ：

* 关键词/embedding 简单聚类检测重叠
* 强相关仓位总和 ≤ 15%
* 定期回测历史持仓在极端情景下的共变

 **动态仓位模型** ：

* 仓位大小 = f(概率优势、流动性深度、NO 价格、样本置信度)
* 示例：
  * NO=0.97 + 良好流动性 + 强边缘 → 小仓（0.5-1%）
  * NO=0.99+ 或 流动性差 → 极小仓或跳过
* **硬上限** ：
* 单仓 ≤ 1.5% 总资金
* 总尾部暴露（所有 NO 仓位） ≤ 40-50%（比 V2 更保守）
* 根据 Kelly 分数或 fractional Kelly 调整，但永不超过硬上限

---

### 5. 退出机制（多触发）

1. **提前止盈** ：NO ≥ 0.995 → 平仓（若流动性允许）
2. **时间退出** ：剩余 < 1 天 或 < 24 小时 → 强制平仓
3. **流动性退出** ：深度显著下降或 spread 过宽 → 逐步减仓
4. **风险退出（新增强化）** ：

* NO 价格反向移动超过 3-5%（视初始价格）
* 分类/主题风险升高或出现异常新闻
* 全局风险指标触发

1. **争议防护退出** ：若市场进入 UMA 争议阶段 → 立即尝试平仓或对冲

---

### 6. 黑天鹅防护系统（核心生存机制）

 **全局风险指标** （实时监控）：

* 持仓中 NO 价格整体下跌比例或速度
* 同类别/主题多市场异常波动
* 平台级代理指标（如加密市场大波动、VIX 类、或全市场 NO 均值变化）
* 近期结算争议事件频率

 **触发条件** （任一满足即动作）：

* 多仓位同时异常
* 连续亏损达到阈值（例如 3 次）
* 全局指标超过历史 95% 分位

 **动作** ：

* 立即停止新开仓
* 强制减仓至总暴露 ≤ 20%
* 进入“冻结模式”（冷却期至少 48 小时）
* 冷却后需人工/回测验证才能恢复

 **额外防护** ：

* 预留缓冲资金（不参与策略）
* 平台风险监控（Polygon、USDC、UMA 相关新闻）

---

### 7. 执行保护与运营要求

* **滑点与深度检查** ：下单前必须验证当前 orderbook 深度支持拟成交规模（含安全缓冲），否则跳过或缩小。
* **部分成交处理** ：自动拆单 + 剩余部分重新评估。
* **API 集成** ：优先使用 Gamma（市场元数据）、CLOB（prices-history、orderbook）、Data API（trades/positions）。
* **混合模式** ：初期所有候选需人工审核分类与异常，成熟后自动化 + 每日人工抽查。
* **日志与仪表盘** ：实时记录所有决策理由、校准数据、暴露分布。

---

### 8. 回测与验证系统（优先级最高，必须先于实盘）

 **目标** ：回答“这个策略在历史上是否真的有正边缘？在哪些条件下存活并盈利？”

 **数据来源** ：

* Gamma API + CLOB prices-history（注意 resolved 市场 granularity 问题，使用 startTs/endTs）
* 社区/开源数据集补充
* 包含已结算市场、price series（关键时间点）、resolution、category

 **回测要求** ：

* 严格时间切片：t 时刻只能使用当时及以前数据
* 模拟完整流程：筛选 → 分类 → 校准 → 风控 → 执行 → 结算 PnL
* 概率校准：分桶 + 类别 + 滚动窗口 + 置信区间
* 压力测试：随机/历史极端情景（连续同类失败、相关性爆发、UMA 式争议模拟、流动性枯竭）
* 参数扫描：NO 阈值、剩余时间、仓位上限、流动性要求等，找出保守最优组合

 **必须输出** （每次回测结构化报告）：

* 总/年化收益率、最大回撤（最重要）、胜率、平均盈亏比、连续亏损次数
* 资金利用率、分类暴露分布
* 样本量警告、禁止区间、高风险类别
* 黑天鹅承受能力（最大可承受连续失败次数而不爆仓）
* 推荐参数 vs 激进参数对比

 **原则** ：若回测最大回撤 > 20-25% 或样本不足导致不可靠 → 进一步收紧参数或暂停该类别。不做未经充分验证的交易。

---

### 9. 风控总原则（写死，不可违反）

1. **生存第一** ：任何时候风险不可控 > 潜在收益
2. **数据驱动** ：不做样本不足或校准不支持的交易
3. **不集中、不长期暴露** ：分散 + 短时限
4. **不相信单一市场概率** ：只相信历史统计边缘 + 置信区间
5. **异常即止** ：系统异常或争议迹象时，优先保住本金
6. **持续迭代** ：每季度复盘回测，更新校准与白名单

---

### 10. 核心监控指标

* 胜率、最大回撤、连续亏损次数
* 资金利用率（目标：不追求高，利用率低也没关系）
* 分类/主题暴露分布
* 校准偏差（实际 vs 隐含）
* 机会缺失天数（反映策略保守程度）
* 执行滑点与部分成交率

---

### 11. 系统本质总结

V3 不是一个高频交易机器人，而是一套 **基于历史验证、动态概率校准、严格分类与流动性约束的尾部风险管理系统** 。

它接受机会稀缺、资金利用率较低的事实，换取更高的生存概率和更小的回撤。

---

### 12. 最关键一句话

> **我们无法消除黑天鹅，但我们可以构建一个系统：只在边缘清晰、可验证、可控时小步参与，并在任何异常信号出现时迅速收缩，确保即使最坏情况发生，系统依然存活并能恢复。**

 **V3 与 V2 主要改进点总结** （便于编辑对比）：

* 更保守的筛选阈值与流动性要求
* 动态滚动校准 + 置信区间（解决样本不足）
* 强化执行保护与退出机制
* 更严格的仓位与暴露上限
* 黑天鹅防护更敏感 + UMA 争议考虑
* 回测优先级最高，增加压力测试细节
* 混合人工/自动模式，降低执行风险
